# PDF Progress Report Redesign

**Date:** 2026-06-04  
**File:** `src/lib/generateReport.js`  
**Status:** Approved for implementation

---

## Problem

The current PDF progress report feels plain and generic. Every step section uses the same blue left-border treatment, there is no logo in the header, and the document has no closing message. It could have been generated by any tool. It does not feel like a FYPro product.

---

## Design Decisions

All decisions were confirmed during brainstorming.

| Decision | Choice |
|---|---|
| Visual direction | Bold Nigerian Tech — high-contrast, confident, Paystack/Mono energy |
| Cover page | No dedicated cover page — bold header leads straight into content |
| Step color identity | Each step gets its own color (see color map below) |
| Typography | Poppins throughout — no DM Serif Display in the PDF |
| Logo | FYPro logo in header and footer |
| Shield watermark | Faint in header background + faint behind closing message block |
| Step number watermark | Large faded digit in background of each step card |
| Companion card treatment | Dashed left border + distinct tint — visually separate from numbered steps |
| Closing message | Dark navy block at the end of the document |

## Component Specs

### Header

```

padding: 20px 24px 18px
```

- **Logo:** `<img src="/fypro-logo-light.png" height="22px">` — left side. Fallback: "FYPro" in white Poppins 800 if image fails to load.
- **Label:** "FYPro · Progress Report" — `rgba(255,255,255,0.65)`, 8px, 700 weight, letter-spacing 2.5px, uppercase.
- **Topic title:** White, 14px, Poppins 700, line-height 1.45. Max-width 65%.
- **Student line:** `rgba(255,255,255,0.7)`, 8.5px, Poppins 500. Format: `Name · Department · Level · University`.
- **Badge (top-right):** "X / 6 COMPLETE" in `#16A34A` pill, JetBrains Mono 7px 700, letter-spacing 1.5px.
- **Date (below badge):** `rgba(255,255,255,0.4)`, JetBrains Mono 7.5px.
- **Shield watermark:** `shield-star.svg` inline, width 120px, fill `#FFFFFF`, `opacity: 0.06`, positioned absolute right `-10px`, vertically centred. Pointer-events none.
- **Background circles:** Two absolute-positioned circle divs at bottom-right and top-right for depth (rgba white, border-radius 50%).

## Step Ordering in the Document

Steps render only if their data is present in state. Order:

1. Step 1 — if `state.stepsCompleted[0]`
2. Step 2 — if `state.stepsCompleted[1] && state.chapterStructure`
3. Companion: Literature Map — if `state.literatureMap`
4. Companion: Abstract Generator — if `state.abstractData`
5. Step 3 — if `state.stepsCompleted[2] && (state.chosenMethodology || state.methodology)`
6. Companion: Instrument Builder — if `state.instrumentBuilder`
7. Step 4 — if `state.stepsCompleted[3] && state.writingPlan`
8. Step 5 — if `state.stepsCompleted[4]`
9. Step 6 — if `state.stepsCompleted[5]`
10. Examiner Questions — if `state.uploadedProject?.reviewData?.examiner_questions?.length`
11. Closing message — always rendered (regardless of completion count)

If `completedCount === 0`: show the "no steps completed yet" placeholder in place of all step content, but still render the closing message.

## Implementation Scope

**File to modify:** `src/lib/generateReport.js` — full rewrite of all HTML builder functions and `buildReportHTML`. The `downloadProgressReport` export function signature and html2pdf configuration stay the same.

**Assets used:**
- `/fypro-logo-light.png` — already loaded via `loadLogoDataUrl()`, keep this
- `shield-star.svg` — inline the SVG path directly in the HTML string (no additional asset loading needed)

**Font loading** — no change. Poppins and JetBrains Mono are already pre-loaded via `document.fonts.load()` before render.

**No new dependencies.** html2pdf.js stays. No changes to `api/`, `vercel.json`, or any other file.

- The html2pdf.js configuration (margins, scale, jsPDF settings, pagebreak mode)

- The filename generation logic (`FYPro-Progress-Report-{slug}.pdf`)
